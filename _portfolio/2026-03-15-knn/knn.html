Breast cancer has become a major public health problem in the current society. The global incidence of breast cancer has been on the rise since the late 1970s. One of the eight women in the United States will have breast cancer in their lifetime. China is not a country with a high incidence of breast cancer, but it should not be optimistic. In recent years, the growth rate of the incidence of breast cancer in China is 1 to 2 percentage points higher than that of high-incidence countries. According to the 2009 breast cancer incidence data released by the National Cancer Centre and the Centre for Disease Control and Prevention of the Ministry of Health in 2012, the incidence of breast cancer ranks first among female malignant tumours in the national tumour registration areas. The total incidence of female breast cancer (rough rate) is 42.55/100,000 in the country, and 51 in cities. .91/100,000, 23.12/100,000 in rural areas.

Early breast cancer detection mainly examines abnormal lumps in breast tissue. If a lump is found, a fine needle aspiration biopsy is needed, and then the cell is examined under the microscope to determine whether the lump is benign or malignant. If it can be modelled through machine learning and automatically diagnosed through the detection data of breast lumps, it will bring great benefits to the medical system. On the one hand, automatic diagnosis can greatly improve the efficiency of detection. On the other hand, the automatic diagnosis combined with a large number of different historical cases can assist doctors in making decisions and reduce the risk of misjudgement.

In this case, based on the public breast cancer diagnosis data, we will use a simple algorithm in machine learning, the K nearest neighbour algorithm, to build an automatic diagnosis model for breast cancer.

1.Data source

We will use the breast cancer diagnostic data set from UCI. The detailed description of the data set can be found at the website: http://archive.ics.uci.edu/ml/datasets/Breast+Cancer+Wisconsin+%28 Diagnostic%29. The breast cancer data includes 569 breast cell biopsy samples, each of which contains 32 variables. Among them, the id variable is the sample identification ID, and the diagnosis variable is the target variable (M stands for malignant and B stands for benign). The other 30 variables are composed of the mean, standard deviation and maximum values of 10 different characteristics of 10 digital nuclei. These 10 basic characteristics are: - radius - texture - perimeter - area - smoothness - compactness （perimeter^2 / area - 1.0） - concavity - concave points - symmetry - fractal dimension

Inconsistent measurement scales of different variables will affect the sample distance calculation in the K nearest neighbour algorithm. For example, if the above three variables are directly involved in the distance calculation, the area_mean variable will have the greatest impact on the distance calculation, which will cause the classification model we build to rely too much on the area_mean variable. Before applying algorithms involving distance calculation such as K nearest neighbour to build a prediction model, we need to standardise the value of variables. Common standardisation methods include min-max standardisation and Z-score standardisation. In this case, we will use the min-max standardisation method to adjust the value of our independent variable to a standard range.

2.1 Data standardisation

In order to standardise the independent variables min-max, we need to implement a min_max_normalise() function. The function input is a numerical vector x. For each value in x, subtract the minimum value of x, and then divide it by the value range of the value in x. The specific implementation code is as follows:

def min_max_normalize(x):
    return (x - x.min())/(x.max() - x.min())

Now, we can standardise the data set with the implemented min_max_normalise() function.

for col in breast_cancer.columns[1:31]:

2.2 Divide the training set and the test set

Due to the lack of such future data, we adopt a simulation method: divide our data set into two parts: the training set and the test set. The training set is used to build a model, while the test set cannot be used in model construction. It is only used to evaluate the performance of the model. In this case, we use 70% (398 samples) of the data to train the model, and 30% (171 samples) to test the model.

3.Model training

For the K nearest neighbour algorithm, the model training stage actually does not include model establishment, and only the training data and its tags need to be stored. In order to use training data for classification, we use the K-neighbour algorithm of the sklearn.neighbors package to implement the KNeighborsClassifier class. For each sample in the test set, a specific distance calculation method will be used to find k nearest neighbours, of which k is a parameter that can be specified. Then, vote through the class labels of these k nearest neighbour samples, so as to classify the test samples. If the number of votes for different classes is equal, the test samples will be randomly classified. Of course, in order to avoid the occurrence of the same number of votes in different classes, k is usually set to an odd number for binary classification problems.

sklern.neighbors.KNeighborsClassifier class has several main parameters, which mean as follows:

  • n_neighbours: k

  • weights: k nearest neighbour voting weight calculation method

  • metric: distance calculation method

For detailed values and setting methods of each parameter, please refer to the website: http://scikit-learn.org/stable/modules/generated/sklearn.neighbors.KNeighborsClassifier.html

Next, we set k to 21 and use the K nearest neighbour algorithm to predict the test data. The prediction results are saved in the breast_cancer_test_pred vector.

from sklearn.neighbors import KNeighborsClassifier
knn_model = KNeighborsClassifier(n_neighbors = 21)
knn_model.fit(breast_cancer_minmax_train, breast_cancer_train_labels)

5.Model performance improvement

For the K nearest neighbour algorithm, the different value selection of K will affect the effect of the model. In addition, different data standardisation methods will also affect the effect of the model. In this section, we will explore whether the change of K value and the change of data standardisation method can improve the prediction results of the model.

5.1 Test the impact of different k values on the model effect

We will test the effect of the model when k = 1, 5, 9, 11, 15, 21, 27 respectively. Because the overall prediction accuracy of the model is already very high. We compare the effects of models under different k values by observing the number and accuracy of False Negative and False Positive.

k_list = (1,5,9,11,15,21,27)
for k in k_list:
    knn_model = KNeighborsClassifier(n_neighbors = k)
    knn_model.fit(breast_cancer_minmax_train, breast_cancer_train_labels)

    breast_cancer_test_pred = knn_model.predict(breast_cancer_minmax_test)
    accuracy = metrics.accuracy_score(breast_cancer_test_labels, breast_cancer_test_pred)
    confusion_mat = metrics.confusion_matrix(breast_cancer_test_labels, breast_cancer_test_pred)
